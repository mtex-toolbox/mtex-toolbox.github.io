--- title: Weighted Burgers Vector Analysis sidebar: documentation_sidebar permalink: WBV.html folder: documentation toc: false --- Weighted Burgers Vector Analysis edit page

The "Weighted Burgers Vector" is a method to analyze geometrically necessary dislocations from EBSD maps, complementary (or alternatively) to GND analysis.

The reference description can be found here: Wheeler et. al., The weighted Burgers vector: a new quantity for constraining dislocation densities and types using electron backscatter diffraction on 2D sections through crystalline materials

It is a moving window technique interpreting the orientation gradient in a given neighborhood and at a given distance to be caused by dislocation lines which crosscut the observation surface. It is weighted/biased by the cosine of the angle between the map-normal the dislocation line direction.

The implementation in MTEX allows for two different methods. The default method is derived from the orientation gradient in x- and y- directions, obtained using a convolution kernel similar to a Prewitt operator, except that the corners are weighted by 0.5. This is comparable to what is termed the "integral method" by other implementations. Here the size of kernel can be adjusted in order to cope with noisy data. Alternatively, there is the "gradient" method, where the WBV is simply derived from the last column of the density tensor| (see GND.html), hereby the gradient is derived from the nearest neighbor pixels. Hence, using high angular resolution EBSD data or denoising of conventional EBSD prior to the analysis is required.

In this section we want to see how we can generally use the EBSD.weightedBurgersVec.html|WBV| function. We will use the same data as in the GND.html demonstration.

{% highlight matlab %} % set up the plotting convention and preference plotx2east setMTEXpref('showMicronBar',0) % import the EBSD data % mtexdata single CS = crystalSymmetry('Fm3m',[4.04958 4.04958 4.04958],'mineral','Al'); ebsd = EBSD.load([mtexDataPath filesep 'EBSD' filesep 'single_grain_aluminum.txt'],... 'CS', CS,'RADIANS','ColumnNames', { 'Euler 1' 'Euler 2' 'Euler 3' 'x' 'y'},... 'Columns', [1 2 3 4 5]); % define the color key ipfKey = ipfHSVKey(ebsd); ipfKey.inversePoleFigureDirection = yvector; % and plot the orientation data plot(ebsd,ipfKey.orientation2color(ebsd.orientations),'micronBar','off','figSize','medium') {% endhighlight %}
{% include inline_image.html file="WBV_01.png" %}

We reconstruct grains because later on, we do not want to compute the WBV across grain boundaries (if present)

{% highlight matlab %} [grains,ebsd.grainId] = calcGrains(ebsd,'angle',2.5*degree,'minPixel',6); % denoise the data % we will use the noisy data later on ebsdN = ebsd.gridify; F = halfQuadraticFilter; ebsd = smooth(ebsd('indexed'),F,'fill',grains); {% endhighlight %}

Computing the WBV

The function expects the input to a gridified EBSD.gridify.html|EBSD|.

{% highlight matlab %} ebsd = ebsd.gridify wbv = weightedBurgersVec(ebsd) {% endhighlight %} {% highlight plaintext %} ebsd = EBSDsquare (y↑→x) Phase Orientations Mineral Color Symmetry Crystal reference frame 0 1 (0.0098%) notIndexed 1 10200 (100%) Al LightSkyBlue m-3m Properties: grainId, oldId, quality Scan unit : um X x Y x Z : [6, 16] x [4, 14] x [0, 0] Normal vector: (0,0,1) Square grid :101 x 101 wbv = vector3d (y↑→x) size: 101 x 101 {% endhighlight %}

The WBV is returned as a vector3d.vector3d.html|vector3d| and we can inspect its magnitude (in 1/scanunit) and direction.

{% highlight matlab %} plot(ebsd,wbv.norm); hold on mtexColorbar mtexTitle('norm of the WBV') {% endhighlight %}
{% include inline_image.html file="WBV_02.png" %}

Visualize the WBV

In order to visualize the direction of the WBV in specimen coordinates, we can use a color key.

{% highlight matlab %} cK = HSVDirectionKey(specimenSymmetry('1')); plot(ebsd,cK.direction2color(wbv),'FaceAlpha',wbv.norm/0.22) mtexTitle('direction of the WBV in specimen coordinates') {% endhighlight %}
{% include inline_image.html file="WBV_03.png" %}
{% highlight matlab %} figure(2) plot(cK) {% endhighlight %}
{% include inline_image.html file="WBV_04.png" %}

We could also display the WBV by small arrows. If we allow for any magnitude, the plot would become quite cluttered. Hence, we will only display those vectors which have a reasonably large magnitude.

{% highlight matlab %} cond = wbv.norm > quantile(wbv.norm,0.85); figure(1), hold on %plot(ebsd,wbv.norm); hold on quiver(ebsd(cond),wbv(cond),'color','k','autoScaleFactor', 2, 'antipodal'); hold off hold off {% endhighlight %}
{% include inline_image.html file="WBV_05.png" %}

If we are simply interested in the distributions of WBV, we can plot them in a spherical projection. Since the WBV is assigned nan at points where it cannot be defined, we need to filter those values

{% highlight matlab %} wbv.antipodal = 0 notNan = ~isnan(wbv); plot(wbv(notNan),'weights',wbv(notNan).norm,'contourf') {% endhighlight %} {% highlight plaintext %} wbv = vector3d (y↑→x) size: 101 x 101 {% endhighlight %}
{% include inline_image.html file="WBV_06.png" %}

The WBV in crystal coordinates

In order to inspect the WBV in crystal coordinates we transform them using the orientations and choose a fitting directional color key

{% highlight matlab %} % transform to crystal reference frame wbvC = inv(ebsd.orientations) .* wbv; % define a directional color key respecting crystal symmetry cKC = HSVDirectionKey(wbvC.CS); % plot the data plot(ebsd,cKC.direction2color(wbvC),'FaceAlpha',wbv.norm/0.22) mtexTitle('direction of the WBV in crystal coordinates') {% endhighlight %}
{% include inline_image.html file="WBV_07.png" %}
{% highlight matlab %} % plot the color key plot(cKC) % overlayed with the hold on plot(wbvC(notNan),'weights',wbvC(notNan).norm,'contour', ... 'contours',0.5:0.25:2,'linecolor','k','ShowText','on', ... 'linewidth',2) hold off {% endhighlight %}
{% include inline_image.html file="WBV_08.png" %}

Effect of windowSize

In case there are reasons why the EBSD data cannot be denoised, the WBV can also be computed with respect to a larger neighborhood. The integer specified with with 'windowSize' gives a 2*n+1 square across which the WBV is computed. The default is a 3-by-3 box.

{% highlight matlab %} close all newMtexFigure('layout',[4,2]) % first we plot again the WBV form the denoised dataset wbv = weightedBurgersVec(ebsd); nextAxis plot(ebsd,wbv.norm); hold on mtexTitle('norm (denoised) / box = 3') nextAxis notNan = ~isnan(wbv); plot(wbv(notNan),'weights',wbv(notNan).norm,'contourf','antipodal') % next we plot the WBV form the noisy dataset for ws = [1 2 3] wbv = weightedBurgersVec(ebsdN,'windowSize',ws); nextAxis plot(ebsdN,wbv.norm); hold on mtexTitle(['WBV norm / box =' num2str(2*ws+1)]) nextAxis notNan = ~isnan(wbv); plot(wbv(notNan),'weights',wbv(notNan).norm,'contourf','antipodal') mtexTitle(['WBV direction / box =' num2str(2*ws+1)]) end mtexColorbar {% endhighlight %}
{% include inline_image.html file="WBV_09.png" %}

Here we see that there is some difference between the noisy and the denoised data in a 3-by-3 and larger neighborhoods. For larger window sizes, we see that there is of course a loss of detail (and empty spaces appear around unfilled points) and a decrease in the norm of WBV, since high orientation gradients are spread by the larger window size. At the other hand, the distribution of the WBV in the pole figures becomes sharper, because there is less scatter in the WBV over larger areas.

Finally, we look at the weighted Burgers vector computed using the gradient method and the denoised data.

{% highlight matlab %} wbv = weightedBurgersVec(ebsd,'gradient'); cK = HSVDirectionKey(specimenSymmetry('1')); plot(ebsd,cK.direction2color(wbv),'FaceAlpha',wbv.norm/0.2) nextAxis notNan = ~isnan(wbv); plot(wbv(notNan),'weights',wbv(notNan).norm,'antipodal','contourf') mtexTitle('WBV direction / gradient' ) colorbar {% endhighlight %}
{% include inline_image.html file="WBV_10.png" %}

We observe that for denoised data the gradient based method results in a more detailed map.
